# Product Guidelines

## Prose & Documentation Style
- Use clear, concise, and plain language
- Write documentation in English
- Keep README and key docs up to date with each feature

## UX Principles
- **Consistency**: Maintain consistent UI patterns across screens and platforms
- **Platform Fidelity**: Respect Android Material Design and iOS Human Interface Guidelines where appropriate
- **Simplicity**: Start simple, add complexity only when needed
- **Responsiveness**: Ensure UI adapts to different screen sizes and orientations

## Branding & Visual Identity
- Use a clean, minimal design language
- Follow Flutter's Material Design 3 defaults unless overridden
- Use a defined color palette and typography scale

## Code & Architecture
- Favor composition over inheritance
- Keep widgets small and focused (single responsibility)
- Use a clear project structure (lib/, test/, etc.)
- Prefer immutability where practical
